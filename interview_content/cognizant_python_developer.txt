1. Q: What is the use of break statement?
   A: Break Statement is a loop control statement that is used to terminate the loop. As soon as the break statement is encountered from within a loop, the loop iterations stop there, and control returns from the loop immediately to the first statement after the loop.

2. Q: What is tuple in Python?
   A: A tuple is created by placing all the items (elements) inside parentheses () , separated by commas. The parentheses are optional, however, it is a good practice to use them.

3. Q: What is an operator in Python?
   A: Operators are special symbols in Python that carry out arithmetic or logical computation. The value that the operator operates on is called the operand.

4. Q: Is Python interpreted language?
   A: For the most part, Python is an interpreted language and not a compiled one, although compilation is a step. Python code, written in . py file is first compiled to what is called bytecode (discussed in detail further) which is stored with a . pyc or.

5. Q: How can we handle URLs in Django?
   A: Django runs through each URL pattern, in order, and stops at the first one that matches the requested URL, matching against path_info . Once one of the URL patterns matches, Django imports and calls the given view, which is a Python function (or a class-based view).

6. Q: Can we make multi-line comments in Python?
   A: There is no such feature as a multi-line comment. # is the only way to comment a single line of code.

7. Q: Which programming language is a good choice between Java and Python?
   A: Java may be a more popular option, but Python is widely used. People from outside the development industry have also used Python for various organizational purposes. Similarly, Java is comparatively faster, but Python is better for lengthy programs.

8. Q: What can we do with the Django framework?
   A: Django applications can contain Python modules, Django-specific modules (views, URLs, models, admin, forms, template tags, etc), static files, templates, database migrations, management commands, unit tests, and more.

üí° Tips
- Have a strong grasp of Python syntax, data types (lists, tuples, dictionaries, sets), control flow (loops, conditionals), and functions.
- Understand classes, objects, inheritance, polymorphism, and encapsulation. Python is an OOP language, and leveraging these concepts is crucial for building robust applications.
- Be proficient in using and creating Python modules and packages for code organization and reusability.
- Familiarize yourself with common data structures like arrays, linked lists, stacks, queues, trees, and graphs.
- Develop problem-solving skills by practicing various algorithms, including sorting, searching, and dynamic programming. This is frequently assessed in technical interviews.
- Gain expertise in essential Python libraries relevant to your role, such as NumPy and Pandas for data manipulation, or requests for web interactions.
- If working on web development, learn frameworks like Django or Flask.
- Depending on the project, delve into libraries for areas like machine learning (Scikit-learn, TensorFlow, PyTorch), data visualization (Matplotlib, Seaborn), or automation.
- Adhere to PEP 8 style guidelines for readable and maintainable code.
- Understand how to write unit tests and effectively debug Python programs using tools like PDB.
