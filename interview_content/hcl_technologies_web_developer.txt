üìå Interview Questions + Answers
    
1. Q: Explain promises in TypeScript.
   A: Promises in TypeScript are objects representing the eventual completion or failure of an asynchronous operation.

2. Q: What is the DOM?
   A: DOM stands for Document Object Model, which is a programming interface for web documents.

3. Q: Explain hooks in JavaScript.
   A:  Hooks in JavaScript are functions that let you use state and other React features without writing a class.

4. Q: What is the event loop in JavaScript?
   A: Event loop in JavaScript is responsible for handling asynchronous operations by executing callback functions in a non-blocking way.

5. Q: How do you handle state in functional components?
   A: State in functional components can be managed using the useState hook in React.

6. Q: What is hoisting in JavaScript?
   A: Hosting in JavaScript refers to the process of declaring a variable or function before it is used in the code.

7. Q: What is the difference between const char* p and char const* p?
   A: Both 'const char* p' and 'char const* p' are equivalent, indicating a pointer to a constant character string.

8. Q: What is a dangling pointer?
   A: A dangling pointer is a pointer that does not point to a valid object or memory location, often due to deallocation.
 
9. Q: Can you explain the basic data types available in C?
   A: C has several data types, including basic types, derived types, and user-defined types, each serving different purposes.

10.Q: What is a function in the C programming language?
   A: A function in C is a block of code that performs a specific task and can be reused throughout a program.

üí° Tips for IBM India
- Use platforms such as freeCodeCamp, The Odin Project, W3Schools, and Udemy to learn or refresh your skills in HTML, CSS, JavaScript, and frameworks.
- Build a portfolio of web development projects to showcase your practical skills. For instance, a simple full-stack project can demonstrate both front-end and back-end knowledge.
- Brush up on Core Java and frameworks like Spring Boot, which are frequently mentioned in HCL job descriptions.
- Be prepared to discuss projects listed on your resume, the technologies used, and the challenges you faced. You may also be given live coding problems or asked scenario-based questions.
- HR Interview: This round evaluates your soft skills, cultural fit, and motivation. Common questions include "Tell me about yourself," "Why HCL?," and discussions on your strengths and weaknesses. 
- The test typically includes a few programming questions. Practice your coding skills on common problems related to data structures and algorithms. 
- Aptitude: Practice quantitative ability (e.g., time and work, percentages), logical reasoning (e.g., puzzles, series), and verbal ability.
- Understand how to optimize page load times and ensure a smooth user experience. This includes techniques like code minification, image optimization, and caching.
- Learn to build and work with RESTful APIs, understanding HTTP methods (GET, POST, PUT, DELETE) and protocols. 
- Study SQL and NoSQL databases. For SQL, practice writing complex queries involving joins and subqueries, and understand concepts like normalization and transactions. For NoSQL, familiarize yourself with databases like MongoDB.

- https://unstop.com/blog/hcl-interview-questions
- Improper version control: Frequent, well-documented commits are essential for team collaboration. Mistakes like infrequent commits, working directly on the main branch, or ignoring merge conflicts can lead to lost work and code instability.
