‚ö†Ô∏è Common Mistakes to Avoid
- Comparing strings with ==: Using == checks if two objects have the same reference, not if their content is the same. Always use the .equals() method for content comparison.
- https://www.quora.com/What-are-some-common-mistakes-newbie-java-programmers-make
- https://www.youtube.com/watch?v=ySdvyCC0Nqo
- Ignoring equals() and hashCode() contracts: When using custom objects in hash-based collections (like HashMap or HashSet), you must override both the equals() and hashCode() methods.
- https://medium.com/javarevisited/the-dark-side-of-java-common-mistakes-that-cost-millions-2145951b070b
- Swallowing exceptions: Using empty catch blocks or broad exception types like catch (Exception e) hides critical bugs and makes your code impossible to debug.
- Modifying a collection during iteration: Attempting to add or remove elements from a collection while iterating over it (with a for-each loop) will cause a ConcurrentModificationException. Instead, use an Iterator and its remove() method for safe modification. 
- https://www.facebook.com/groups/356484977765272/posts/6864332173647154/
- Using floating-point types for currency: Using float or double for monetary calculations can cause precision errors. Use BigDecimal for accurate and precise decimal arithmetic.
