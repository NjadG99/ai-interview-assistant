1. Q: What is DevOps methodology?
   A: DevOps methodology integrates development and operations to enhance collaboration, efficiency, and software delivery speed.

2. Q: What is agile methodology?
   A: Agile methodology is an iterative approach to software development that emphasizes flexibility, collaboration, and customer feedback.

3. Q: What is a public cloud?
   A: A public cloud is a computing model where services are offered over the internet to multiple users, ensuring scalability and cost-effectiveness.

4. Q: Write Terraform code to create an S3 bucket.
   A: Terraform code to create an S3 bucket in AWS with versioning and lifecycle policies.

      -Use the 'aws_s3_bucket' resource to define the S3 bucket.
      -Enable versioning by setting 'versioning' block to true.
      -Add lifecycle rules to manage object expiration.
      -Example: 'lifecycle_rule' block can delete objects older than 30 days.

5. Q: What are the different S3 storage classes?
   A: Amazon S3 offers storage classes to optimize cost and performance:

      -S3 Standard**: For frequently accessed data.
      -S3 Intelligent-Tiering**: Auto-adjusts tiers based on usage.
      -S3 Standard-IA**: For infrequently accessed data needing quick retrieval.

   A:  I have experience with tools like Jenkins, Docker, Kubernetes, and Terraform, enhancing CI/CD and infrastructure management.

8. Q: Explain the Kubernetes Architecture.
   A: Kubernetes is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications.

9. Q: What are ConfigMaps in Kubernetes?
   A: ConfigMaps in Kubernetes are used to store configuration data in key-value pairs and can be used by pods to access configuration information.

10.Q: Can you explain in detail about clusters creation and how you handled practical bugs?
   A: Clusters creation involves setting up multiple servers to work together, while handling practical bugs requires troubleshooting and fixing issues in the cluster.
       -Clusters creation involves setting up multiple servers to work together to distribute workload         and increase reliability.
       -Tools like Kubernetes or Docker Swarm can be used to create and manage clusters efficiently.
