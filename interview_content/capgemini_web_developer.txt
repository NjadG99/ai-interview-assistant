1. Q: What is the difference between client-side rendering and server-side rendering?
   A: -Client-side rendering (CSR): The server sends a minimal HTML shell and JavaScript loads data and generates the page in the browser. Good for dynamic single-page apps, but initial load might be slower.
      -Server-side rendering (SSR): HTML is generated on the server and sent fully to the client. Faster first-page load, better SEO.

2. Q: What is a pure function? Why is it useful?
   A: A pure function is a function which:

Always returns the same output given the same input.

Has no side effects (doesn‚Äôt modify external state, doesn‚Äôt depend on external mutable data).

3. Q: What is a closure and where is it used?
   A: A closure is when an inner function has access to variables from its outer (enclosing) function even after the outer function has returned. It ‚Äúcloses over‚Äù the lexical environment. It‚Äôs often used for encapsulation, callbacks, private variables, etc.

4. Q: How do async/await and promises work?
   A: A promise represents a value that may be available now, or in the future, or never. async/await is syntax sugar over promises ‚Äî async functions return a promise, await pauses until the promise resolves (or rejects). Underneath, it‚Äôs non‚Äëblocking and handled via promise chaining.

5. Q: Explain the event loop in JS.
   A: JavaScript uses a single-threaded event loop. The call stack executes synchronous code. When asynchronous things (timers, I/O, promises) complete, their callbacks go to a task queue (or microtask queue). The event loop picks tasks from those queues when the stack is free.

6. Q: What is REST?
   A: REST is an architectural style for APIs using stateless communication over HTTP. Common methods: GET, POST, PUT, PATCH, DELETE. Use status codes (200, 201, 204, 400, 404, 500, etc.).

7. Q: What are indexes in database? Why use them?
   A: Indexes speed up query lookups by providing efficient data access (e.g. B‚Äëtree). But they slow writes/inserts and take storage. Use indexes on columns often used in WHERE, JOIN, ORDER BY.

8. Q: Explain the Redux life cycle.
   A: Redux life cycle involves actions, reducers, store, and state management in a predictable flow.

9. Q: Explain the Context API.
   A: The Context API in React allows for state management and sharing data across components without prop drilling.

10.Q: What are the key features and benefits of React.js as a frontend library?
   A: React offers a component-based architecture, efficient rendering, and a strong ecosystem, making UI development faster and more maintainable.
