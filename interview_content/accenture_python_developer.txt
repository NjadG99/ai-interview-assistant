2. Q: In a Python Application, How to find Bugs or perform Static Analysis?

3. Q: When is Python Decorator used?
   A: A decorator is a design pattern in Python that allows a user to add new functionality to an existing object without modifying its structure. Decorators are usually called before the definition of a function you want to decorate.

- Modifying a list while iterating: Changing the size of a list while looping through it can cause unintended behavior and skip elements. Use a new list or a list comprehension instead.
- Using mutable objects as default arguments: Because Python evaluates default arguments only once when the function is defined, a mutable default (like a list or dictionary) is shared across all function calls. Instead, set the default to None and initialize the object inside the function.
- Catching generic exceptions: A bare except: can hide critical bugs and make debugging much harder. It's best to catch specific exceptions (e.g., ValueError, FileNotFoundError) to handle errors appropriately.
- Ignoring PEP 8 style guidelines: Inconsistent and unreadable code creates friction in a team environment. Use a linter or code formatter (like Black) to automatically enforce a consistent style.
- Using inefficient data structures: Choosing the wrong data structure, such as a list for fast membership checks instead of a set, can significantly impact performance on large datasets. 
- Neglecting testing: Skipping or under-prioritizing testing leads to lower code quality and more bugs in production.
