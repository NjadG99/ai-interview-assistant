6. Q: What is the difference between list and tuple in Python?
   A: Lists and tuples are data structures in Python whereas the main difference is lists are mutable while tuples are immutable. A list has a variable size while a tuple has a fixed size.

7. Q: What are the global and local variables in Python?
   A: There are two types of variables: global variables and local variables. The scope of global variables is the entire program whereas the scope of local variable is limited to the function where it is defined.

‚ö†Ô∏è Common Mistakes to Avoid
- Incorrect Indentation: Python relies on indentation for code blocks. Inconsistent or incorrect indentation leads to IndentationError and makes code difficult to read.
- Neglecting Code Readability: Writing cryptic or overly complex code without comments or clear variable names hinders understanding and maintenance.
- Not Using Docstrings: Failing to document functions, classes, and modules with docstrings makes it challenging for others (and your future self) to understand their purpose and usage.
- Inconsistent Naming Conventions: Not adhering to PEP 8 guidelines for variable, function, and class naming (e.g., snake_case for variables/functions, CamelCase for classes) reduces consistency.
- Not Handling Exceptions: Failing to implement try-except blocks for potential errors (like ZeroDivisionError, FileNotFoundError) can lead to program crashes.
- Overusing Global Variables: Excessive use of global variables can make code harder to reason about, test, and debug.
- Ignoring Computational Complexity: Not considering the time and space complexity of algorithms can lead to performance bottlenecks, especially with large datasets.
- Using Inefficient Data Structures: Choosing the wrong data structure for a task (e.g., using a list for frequent lookups instead of a dictionary or set) can impact performance.
- Hiding Errors: Catching broad exceptions without specific handling or logging can mask underlying issues, making debugging difficult.
- Modifying Lists While Iterating: Modifying a list while iterating over it can lead to unexpected behavior and bugs. Consider creating a new list or using list comprehensions.
