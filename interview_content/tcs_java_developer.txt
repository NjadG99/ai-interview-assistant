5. Q: Explain the concepts of OOP.
   A: OOP (Object-Oriented Programming) is a programming paradigm based on objects that encapsulate data and behavior.

Encapsulation: Bundling data and methods that operate on the data within one unit (e.g., a class).

Inheritance: Mechanism to create a new class from an existing class, inheriting its properties (e.g., a 'Dog' class inheriting from an 'Animal' class).

Polymorphism: Ability to present the same interface for different underlying forms (e.g., method overloading and overriding).

9. Q: What is the Java Virtual Machine (JVM)?
   A: The Java Virtual Machine (JVM) is a software-based engine that runs Java programs. It is part of the Java Runtime Environment (JRE) and serves as an abstract computing machine that enables a computer to run a Java program.

10.Q: Why is Java a Platform-Independent Language?

- Common mistakes for TCS Java developers include poor exception handling, neglecting input validation and security, misusing static variables and concurrency, inefficiently handling memory and performance.
- Incorrectly using try-catch blocks or swallowing exceptions with empty blocks can lead to resource leaks and unstable applications. 
- Trying to use an uninitialized object reference can be avoided by initializing objects before use or utilizing the Optional class. 
- Overusing static members can lead to tight coupling, making code harder to test and maintain. 
- Overlooking security can lead to SQL injection and other attacks; always validate and sanitize user input and implement robust authentication and authorization. 
- kipping unit and integration tests can result in unreliable code; use tools like JUnit and TestNG for robust testing. 
- Failing to validate and sanitize user input creates security risks like SQL injection and cross-site scripting (XSS). 
- The tech landscape evolves quickly; failing to learn new features, languages, and frameworks can lead to obsolescence.
